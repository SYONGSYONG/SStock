# 05-tasks.md — 진행 체크리스트

## 진행 규칙

> 아래 규칙을 위반하면 작업을 중단하고 원인을 해결한 뒤 재진행한다.

1. **순서 엄수** — Phase·단계 번호 순서대로 진행. 건너뜀 금지.
2. **단계별 검증 필수** — 검증 방법을 실제로 실행해 통과를 확인한 후에만 완료(✅) 체크.
3. **테스트 동반** — 각 기능 단계는 테스트와 함께 완료한다(절대규칙 3).
4. **안전 우선** — 실전 주문 경로는 가드 테스트가 통과하기 전까지 활성화하지 않는다.

상태 표기: ⬜ 미착수 / 🟡 진행중 / ✅ 완료

목표: MVP에서 "후속 보강"으로 미룬 항목을 단계적으로 구현. **각 항목은 독립적이며 우선순위에 따라 선택 진행한다.**

> Phase 1~5(MVP)는 완료 상태다. 아래 항목들은 02-specs/01-product에 명세돼 있으나
> MVP 범위 밖으로 미뤄둔 보강 기능이다. `constants.py`에 TR_ID만 선언돼 있고 호출
> 로직은 미구현 상태다.

| # | 단계 | 검증 | 상태 |
|---|------|------|------|
| 1 | 분봉/일봉 차트 (`inquire-time-itemchartprice` 등) | REST 차트 조회 + 프론트 차트 렌더 | ⬜ |
| 2 | 호가창 10단계 (`inquire-asking-price`, `H0STASP0`) | 호가 조회/구독 테스트 + 프론트 호가표 | ⬜ |
| 3 | 매수가능조회 사전 점검 (`inquire-psbl-order`) | 주문 전 매수가능 금액/수량 확인 + 가드 연동 테스트 | ⬜ |
| 4 | 실시간 체결통보 (`H0STCNI0`, AES-CBC 복호) | 체결 수신 파싱 테스트 + 포지션 즉시 갱신 | ⬜ |
| 5 | 주문 정정 (`order-rvsecncl`, 정정 구분 01) | 정정 API mock 테스트 | ⬜ |
| 6 | 종목별 금액 한도 내 자동 수량 산정 | `max_amount` 기반 매수 수량 계산 테스트 | ⬜ |

> 참고: 현재 종목별 한도(`max_qty`/`max_amount`)는 **초과 시 거부**(01-product 명세대로)로
> 동작한다. 6번은 "한도 내에서 최대 수량을 자동 매수"하도록 확장하는 별도 보강 항목이다.
