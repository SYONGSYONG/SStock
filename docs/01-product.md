# 01-product.md — 제품 정의

## 제품 목표

1. **실시간 시세 모니터링** — 관심 종목의 현재가·호가·체결을 KIS 웹소켓으로 실시간 수신하여 대시보드에 표시한다.
2. **룰 기반 자동 매매** — 설정한 지표 규칙(이동평균 돌파, RSI 등)에 따라 매수/매도 신호를 생성하고 주문을 자동 집행한다.
3. **안전한 운영** — 기본은 모의투자. 봇 시작/정지, 일일 한도, 전 거래 기록으로 사고를 방지한다.

## MVP 범위

### 기능 1 — 실시간 시세 모니터링

- 관심 종목 목록 등록/삭제 (종목코드 6자리)
- KIS 웹소켓 실시간 체결가(`H0STCNT0`)·호가(`H0STASP0`) 구독
- 대시보드에 종목별 현재가·등락률·거래량·호가 표시
- 시세 끊김(웹소켓 재연결) 자동 복구 + 연결 상태 표시
- 분봉/일봉 차트 (REST `inquire-time-itemchartprice` / `inquire-daily-itemchartprice`)

### 기능 2 — 룰 기반 전략 엔진

- 전략을 **설정값(JSON/DB)**으로 정의 — 코드 수정 없이 파라미터 변경
- MVP 내장 규칙:
  - 이동평균 골든/데드 크로스 (단기/장기 기간 설정)
  - RSI 과매수/과매도 (기준선 설정)
- 종목별로 전략 on/off, 주문 수량·금액 한도 설정
- 신호 발생 시 매수/매도 **신호 로그** 생성 (집행과 분리하여 추적)

### 기능 3 — 자동 주문 집행 (모의투자)

- 신호 → 주문 변환: 매수가능조회/매도가능수량조회로 검증 후 주문
- 주식주문(현금) `order-cash` 호출 (모의 TR_ID 사용)
- 체결 통보(`H0STCNI0`) 수신 → 포지션·잔고 갱신
- 정정/취소 (`order-rvsecncl`) 지원
- **안전 가드**: 일일 최대 주문 횟수·금액, 종목당 최대 보유, 봇 정지 시 신규 주문 차단

### 기능 4 — 봇 제어 + 모니터링 대시보드

- 봇 **시작/정지** 토글 (정지 시 자동 주문 전면 중단, 시세 수신은 유지)
- 현재 보유 포지션·평가손익 (`inquire-balance`)
- 신호 로그 / 주문·체결 로그 / 시스템 로그 뷰어
- 모의/실전 **모드 표시 배너** (실전일 때 강한 경고색)

## 종목별 자본 칸막이 (capital envelope)

종목마다 투입할 **원금 한도**를 설정해 자금을 칸막이로 나눈다. 한도는 보수적으로
원금을 기준으로 하되, 손실이 나면 줄여 위험을 제한한다.

- **한도 = 원금(principal) + min(0, 실현손익)** — 평균원가법으로 매도 확정 손익만 반영.
- 매수는 `보유원가 + 주문금액 ≤ 한도`일 때만 허용(초과 시 안전가드가 거부, `ENVELOPE_EXCEEDED`).
- **실현이익은 한도를 키우지 않는다**(종이이익으로 위험 확대 방지). 포지션을 모두 정리하면
  한도는 원금으로 돌아오고, 실현이익은 정보(누계)로만 표시한다.
- **실현손실은 한도를 줄인다**(손실 후 재진입 위험 축소, 안전장치).
- 단위는 **종목별**(한 종목을 거래하는 여러 전략이 같은 칸막이를 공유). 미설정 종목은 칸막이 제한 없음.
- 매도는 칸막이 제한을 받지 않는다(자금 회수 방향).
- API: `GET/PUT /api/budgets`, `DELETE /api/budgets/{symbol}`. 가드는 `risk_guard.check_order`에 통합.

### 계좌 주문가능현금과의 관계 (인터락 없음, 표시+경고만)

칸막이 원금 합계가 **계좌 주문가능현금을 초과하지 못하게 막는 하드 인터락은 없다.**
칸막이는 종목별 *논리적* 자금 분할이며, 합계가 실제 현금을 넘어도 설정 자체는 허용한다
(모의투자 연습·전략 실험의 유연성 유지). 대신 칸막이 패널에 **설정가능금액**을 표시해
사용자가 과다 배정을 인지하도록 한다.

- **설정가능금액 = 주문가능현금(`/api/account/balance`의 `orderable_cash`) − Σ(각 칸막이 가용액)**.
- 설정가능금액이 음수이거나, 입력 원금이 설정가능금액을 초과하면 **경고색으로 표시**(차단하지 않음).
- 실제 현금 부족 시점의 차단은 KIS 주문 API(집행 단계)가 담당한다.

## 명시적 MVP 외 범위 (구현하지 않음)

- 실전 자금 자동매매의 무인 장기 운용 (MVP는 감독 하 사용)
- 다중 사용자·계좌, 인증/회원 관리
- 백테스팅 엔진 (별도 Phase로 분리)
- 해외주식·파생·ELW
- 머신러닝 기반 전략
- 모바일 앱 (MVP는 데스크톱 브라우저)
