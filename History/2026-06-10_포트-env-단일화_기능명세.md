# 기능 명세 — 서버/프론트 포트를 저장소 루트 `.env` 단일 출처로 이전

> **목적**: 흩어져 하드코딩된 서버·클라이언트 포트/호스트를 **저장소 루트 `.env` 한 곳**에서
> 정의하고, 모든 기동 스크립트와 빌드 설정이 그 값을 읽도록 통일한다. 포트를 바꿀 때
> **단 한 파일(`.env`)만** 수정하면 전체가 따라오게 만든다.
>
> 이 문서는 **다른 세션·다른 프로젝트에서 동일 기능을 재현**하기 위한 기능 명세다.
> SStock(FastAPI + Vite/React, Windows/PowerShell) 기준으로 작성됐으나, 패턴 자체는
> 스택 무관하게 적용 가능하다. 최초 적용일: 2026-06-10 (SStock).

---

## 1. 해결하려는 문제 (Why)

같은 포트 번호가 여러 파일에 **중복 하드코딩**되어 있으면, 포트를 바꿀 때 한 군데라도
빠뜨려 "프론트는 떴는데 API가 안 붙음" / "stop이 프로세스를 못 죽임" 같은 불일치가 발생한다.
SStock 최초 상태에서 포트가 박혀 있던 위치(예시):

- 백엔드 8000: `start_backend.bat`, `scripts/start-server.ps1`(실행·포트감지·표시), vite 프록시 대상, `backend/.env`(미사용 상태로 죽어 있던 `HOST`/`PORT`)
- 프론트 5173: `vite.config.ts`, `scripts/start-server.ps1`/`stop-server.ps1`, `start_frontend.bat`

→ **단일 출처(Single Source of Truth)** 부재가 핵심 문제. 이를 루트 `.env`로 모은다.

---

## 2. 설계 (What)

### 2.1 단일 출처: 저장소 루트 `.env`

루트에 **포트/호스트 전용 `.env`**(시크릿 없음)를 둔다. 키 스키마:

| 키 | 기본값 | 의미 |
|----|-------|------|
| `BACKEND_HOST` | `127.0.0.1` | 백엔드 바인딩 호스트. 보통 루프백 유지(외부 노출 불필요 — 프론트가 프록시로 전달) |
| `BACKEND_PORT` | `8000` | 백엔드 포트. 프론트 프록시 대상(`/api`·`/ws`)도 이 값을 따라감 |
| `FRONTEND_HOST` | `127.0.0.1` | `true`면 `0.0.0.0` 바인딩(같은 네트워크 다른 PC 접속 허용), `127.0.0.1`이면 로컬만 |
| `FRONTEND_PORT` | `8001` | 프론트(dev 서버) = 브라우저 접속 포트 |

**원칙**
- `.env`에는 **시크릿을 넣지 않는다**(포트/호스트만). 시크릿은 기존 `backend/.env`(별도, git 제외)에 유지.
- 모든 소비자는 `.env` 미존재 시 **기본값으로 동작**(클론 직후에도 그냥 돌아감).
- `.env`는 git 제외, `.env.example`(템플릿)은 git 추적.

### 2.2 소비자(consumer) 목록

1. **빌드 설정(vite)** — 프론트 host/port + 백엔드 프록시 대상
2. **통합 기동 스크립트(PowerShell)** — 백엔드 실행 인자·포트 감지·안내
3. **통합 종료 스크립트(PowerShell)** — 종료 대상 포트
4. **개별 기동 배치(.bat)** — 가시 창 실행용
5. **문서(docs)** — 포트 설정 안내
6. **백엔드 설정 정리** — 중복되던 `HOST`/`PORT` 제거

## 3. 구현 (How) — 단계별

### 단계 0. `.gitignore` 확인
루트 `.env`가 무시되고 `.env.example`은 추적되는지 확인. 보통 아래 패턴이면 충족:
```gitignore
**/.env
**/.env.*
!**/.env.example
```
검증: `git check-ignore .env` → 출력되면 무시됨(OK). `git check-ignore .env.example` → 출력 없으면 추적됨(OK).

### 단계 1. 루트 `.env.example`(커밋) + `.env`(로컬) 생성
`.env.example`:
```dotenv
# ── 네트워크/포트 (단일 출처) ──
# 기동 스크립트와 vite가 모두 읽는다. 포트 변경은 여기 한 곳만.
BACKEND_HOST=127.0.0.1
BACKEND_PORT=8000
# FRONTEND_HOST=true → 0.0.0.0(LAN 접속 허용), 127.0.0.1 → 로컬만(보안 기본값)
FRONTEND_HOST=127.0.0.1
FRONTEND_PORT=8001
```
`.env`는 같은 키에 실제 운영값(예: LAN 노출하려면 `FRONTEND_HOST=true`).

### 단계 2. Vite 설정 — 루트 `.env`를 읽는 함수형으로
`vite.config.ts`는 `defineConfig(({ mode }) => …)` 함수형 + `loadEnv(mode, repoRoot, "")`로
**상위(저장소 루트) `.env`**를 읽는다. (프론트 디렉터리에서 실행되므로 `process.cwd()/..` = 루트)
```ts
import { resolve } from "node:path";
import { defineConfig, loadEnv } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig(({ mode }) => {
  const repoRoot = resolve(process.cwd(), "..");
  const env = loadEnv(mode, repoRoot, ""); // prefix="" → 모든 키(시크릿 없음)
  const backendHost = env.BACKEND_HOST || "127.0.0.1";
  const backendPort = env.BACKEND_PORT || "8000";
  const frontendPort = Number(env.FRONTEND_PORT || "8001");
  const frontendHost: string | boolean =
    env.FRONTEND_HOST === "true" ? true : env.FRONTEND_HOST || "127.0.0.1";
  return {
    plugins: [react()],
    server: {
      host: frontendHost,
      port: frontendPort,
      strictPort: true, // 포트 고정(폴백으로 인한 stop 누락 방지)
      proxy: {
        "/api": { target: `http://${backendHost}:${backendPort}`, changeOrigin: true },
        "/ws":  { target: `ws://${backendHost}:${backendPort}`,  ws: true },
      },
    },
    test: { /* … 기존 vitest 설정 유지 … */ },
  };
});
```
- **`loadEnv`의 envDir를 루트로** 지정하는 게 핵심. prefix `""`는 비-`VITE_` 키도 (서버측 config 용도로) 읽기 위함 — 이 값들은 클라이언트 번들에 노출되지 않는다.
- 시크릿이 든 `backend/.env`는 **읽지 않는다**(envDir가 루트라 자동 분리).

### 단계 3. 통합 기동 스크립트(PowerShell) — `.env` 파서 추가
스크립트 상단에서 루트 `.env`를 파싱해 변수화(기본값 fallback 포함). PowerShell 5.1 호환을 위해
**ASCII 주석**만 사용.
```powershell
$cfg = @{ BACKEND_HOST='127.0.0.1'; BACKEND_PORT='8000'; FRONTEND_PORT='8001' }
$envFile = Join-Path $root '.env'
if (Test-Path $envFile) {
    foreach ($line in Get-Content $envFile) {
        if ($line -match '^\s*#') { continue }
        if ($line -match '^\s*([A-Za-z_]+)\s*=\s*(.+?)\s*$') { $cfg[$matches[1]] = $matches[2] }
    }
}
$bHost = $cfg['BACKEND_HOST']; $bPort = $cfg['BACKEND_PORT']; $fPort = $cfg['FRONTEND_PORT']
```
이후 사용:
- 백엔드 실행: `Start-Process … -ArgumentList '-m','uvicorn','app.main:app','--host',$bHost,'--port',$bPort`
- 포트 감지: `Test-Port ([int]$bPort)`, `Test-Port ([int]$fPort)`
- 안내 메시지에 `${bHost}:${bPort}`, `localhost:${fPort}` 사용.

### 단계 4. 통합 종료 스크립트(PowerShell) — 종료 포트도 `.env` 기반
같은 파서로 읽어 종료 대상 포트 배열 구성:
```powershell
$ports = @([int]$cfg['BACKEND_PORT'], [int]$cfg['FRONTEND_PORT'])
```
(포트로 죽이는 것 + 저장소 디렉터리 기준 python/node 정리를 병행하면 고아 프로세스까지 안전하게 종료.)

### 단계 5. 개별 기동 배치(.bat) — `for /f`로 `.env` 파싱
`%~dp0`는 배치 위치(저장소 루트) + 끝에 `\` 포함 → `%~dp0.env`가 루트 `.env`의 절대경로.
```bat
set "BACKEND_HOST=127.0.0.1"
set "BACKEND_PORT=8000"
if exist "%~dp0.env" (
  for /f "usebackq eol=# tokens=1,2 delims==" %%a in ("%~dp0.env") do (
    if /i "%%a"=="BACKEND_HOST" set "BACKEND_HOST=%%b"
    if /i "%%a"=="BACKEND_PORT" set "BACKEND_PORT=%%b"
  )
)
… --host %BACKEND_HOST% --port %BACKEND_PORT%
```
- `eol=#` → `#`로 시작하는 주석 줄 무시. `delims==` + `tokens=1,2` → `KEY=VALUE` 분해.
- 프론트 배치도 동일 패턴으로 `FRONTEND_PORT`만 읽어 안내에 사용(vite 자체는 `.env`를 직접 읽음).

### 단계 6. 백엔드 설정 중복 제거
- `backend/.env` / `backend/.env.example`의 `HOST`/`PORT` 제거(루트 `.env`로 이전했음을 주석으로 남김).
- 설정 모델(pydantic-settings 등)에서 **아무 데서도 참조되지 않는** `host`/`port` 필드 제거.
  - 사전 확인: `grep -rn "settings\.host\|settings\.port" app/ tests/` 결과가 비어 있어야 안전.
  - `extra="ignore"`면 `.env`에 잔존 키가 있어도 무해하므로, 필드 제거가 동작을 깨지 않는다.

### 단계 7. 문서 갱신
`docs`에 "포트/호스트 설정 — 루트 `.env` 단일 출처" 절(키 표 + 기본값 + LAN 접속·보안 주의) 추가.

5. **실기동**: stop → start 후 `Test-Port`가 OK 보고, 브라우저 `http://localhost:<FRONTEND_PORT>` 접속.

## 5. 주의사항 / 엣지케이스

- **`127.0.0.1` vs `localhost`**: `127.0.0.1`은 IP(IPv4 확정), `localhost`는 이름→IPv4(`127.0.0.1`) 또는 **IPv6(`::1`)**로 해석될 수 있다. 백엔드를 `--host 127.0.0.1`로 IPv4만 바인딩했는데 클라이언트가 `localhost`를 `::1`로 풀면 연결 실패. 그래서 **프록시 대상은 `127.0.0.1` IP로 명시**(이름 해석 불일치 차단).
- **다른 PC 접속 = 프론트만 열면 됨**: 프록시는 호스트(이 PC) 내부에서 서버사이드로 백엔드에 전달하므로, **백엔드는 `127.0.0.1` 유지**해도 다른 PC가 `프론트:포트`에 접속하면 동작. 백엔드 직접 호출이 필요할 때만 `BACKEND_HOST=0.0.0.0`.
- **LAN 노출 보안**: 봇/대시보드를 `FRONTEND_HOST=true`로 열면 같은 네트워크 누구나 접근(인증 없으면 위험). 신뢰망 한정, 인터넷 직접 노출(포트포워딩) 금지. 방화벽 인바운드 허용 필요.
- **`strictPort: true`**: 폴백 포트 금지 → stop 누락 방지. 단, 해당 포트가 점유돼 있으면 기동 실패(의도된 동작).
- **PowerShell 5.1**: `.ps1`은 **ASCII 주석**(비-ASCII는 BOM 없으면 깨질 수 있음). `2>&1` 네이티브 리다이렉트 주의.
- **배치 인코딩**: `.env` 주석에 한글이 있어도 데이터 줄(KEY=VALUE)은 ASCII면 `for /f` 파싱 무해(주석은 `eol=#`로 skip).
- **CRLF 경고**: Windows에서 `git add` 시 LF→CRLF 경고는 정규화일 뿐 무해.
- **시크릿 분리**: 포트 `.env`에 시크릿 금지. vite `loadEnv`가 시크릿 든 `backend/.env`를 읽지 않도록 envDir를 루트로 둔다.

## 6. 재사용 체크리스트 (다른 세션에서)

- [ ] 루트 `.env.example`(커밋) + `.env`(로컬) 생성, 4개 키 정의
- [ ] `.gitignore`로 `.env` 제외 / `.env.example` 추적 확인
- [ ] 빌드 설정(vite 등)이 루트 `.env`를 읽도록 수정(프록시 대상 포함)
- [ ] 통합 기동/종료 스크립트가 `.env` 파싱(기본값 fallback)
- [ ] 개별 기동 스크립트(.bat 등)도 `.env` 파싱
- [ ] 기존에 포트가 박혀 있던 **모든** 위치 grep으로 색출 후 제거(`grep -rn "<옛포트>"`)
- [ ] 백엔드 설정의 중복/미사용 host·port 필드 제거(참조 grep 확인 후)
- [ ] 문서에 포트 설정 절 추가
- [ ] 검증 5종(gitignore/vite실값/PS파서/타입·테스트/실기동) 통과
- [ ] 보안: LAN 노출 여부·방화벽·시크릿 분리 점검
