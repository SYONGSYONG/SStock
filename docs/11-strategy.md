# 11. 전략 (매매 규칙 · 지표 · 엔진)

SStock의 **룰 기반 자동매매 전략**을 한곳에 정리한 단일 문서다. 제품 정의(01)·구현 명세(02)에
흩어져 있던 전략 내용을 여기로 모았다. 전략을 추가·수정할 때는 **이 문서를 먼저 갱신**한다.

> 정본 코드 위치
> - 전략 로직: `backend/app/strategies/ma_cross.py`, `rsi_ma.py`
> - 지표: `backend/app/strategies/indicators.py` (`sma`/`rolling_sma`/`rsi`/`to_tick_bars`/`closed_ticks`)
> - 등록/생성: `backend/app/strategies/registry.py`, 공통 인터페이스 `base.py`(`Signal`)
> - 사용자용 도움말·파라미터·비교표(UI 단일 진실): `frontend/src/lib/strategy.ts`
> - 봇 연동(신호→주문): `backend/app/bot/trading_bot.py`

---

## 개요 — 전략 엔진이 하는 일

- 전략은 **설정값(DB `strategy_config.params`, JSON)**으로 정의한다 — 코드 수정 없이 파라미터 변경.
- 종목별로 전략 **on/off**, 주문 **수량·금액 한도**를 설정한다(`max_qty`/`max_amount`).
- 신호 발생 시 **신호 로그(`signals`)**를 남기고, 집행(주문)과 **분리**해 추적한다.
- 전략은 **무상태(stateless)**다 — 매 틱 `trading_bot.on_tick`이 종가 히스토리를 통째로 넘겨
  전략 객체를 새로 만들어 평가한다. 자세한 구조·성능은 [엔진 성능](#지표-계산--엔진-성능) 참고.
- **확정봉만 평가 + 직전 방향 신호 억제**로 교차 1회당 신호 1건을 보장한다([신호 억제](#틱봉--확정봉--신호-중복-억제)).

현재 MVP 내장 전략은 **2종**이다: 이동평균 크로스(`ma_cross`), RSI + MA 필터(`rsi_ma`).

---

## 전략 1 — 이동평균 크로스 (`ma_cross`)

빠른 단기선과 느린 장기선이 **교차하는 순간**을 추세 전환 신호로 보는 **추세추종** 전략.

**매매 규칙** (틱봉 종가 기준, 확정봉만):
- **골든크로스**: 단기선(MA단기)이 장기선(MA장기)을 **아래→위** 돌파 → **매수**(상승 전환)
- **데드크로스**: 단기선이 장기선을 **위→아래** 이탈 → **매도**(하락 전환)
- 두 선이 **교차하는 그 순간에만** 신호가 나며, 한쪽이 위에 머무는 동안엔 신호가 없다(엣지 트리거).

**파라미터**

| 키 | 라벨 | 기본값 | 범위 | 의미 |
|----|------|-------|------|------|
| `short` | 단기 | 5 | 1~999 | 틱봉 `short`개 평균선(MA, 빠르게 반응). `short < long` 필수 |
| `long` | 장기 | 20 | 2~999 | 틱봉 `long`개 평균선(MA, 느리게 반응 = 추세) |
| `bar_ticks` | 틱봉 | 50 | 1~200 | 원시 틱 `bar_ticks`개를 한 봉으로 묶어 그 종가로 계산(클수록 노이즈↓) |

**워밍업**: 50틱봉 × 장기 20이면 첫 신호까지 약 **1,050틱** 필요. 봇 재시작 시 다시 누적한다.

**구현 메모**: 이동평균은 `rolling_sma`(러닝 합계)로 계산하며 일반 SMA와 동일하다. 판정은
`prev_diff <= 0 < cur_diff`(매수) / `prev_diff >= 0 > cur_diff`(매도)로 마지막 두 확정봉의 부호
전환을 본다.

## 전략 2 — RSI + MA 필터 (`rsi_ma`)

추세 필터(MA)로 매수 방향을 거른 뒤 RSI로 진입 타이밍을 잡고, 추세가 깨지면 즉시 빠져나오는
전략. "**상승추세일 때만 눌린 자리에서 사고, 과열되거나 추세가 무너지면 판다**"가 핵심.

**매매 규칙** (틱봉 종가 기준, 확정봉만):
- **매수**: 현재가가 추세선(MA) **위**(상승추세) **그리고** RSI가 과매도선(`low`)을 **아래→위** 회복
- **매도①**: RSI가 과매수선(`high`)을 **위→아래** 이탈(과열 청산)
- **매도②(안전장치)**: 현재가가 MA를 **위→아래** 하향 돌파(추세 이탈 — 그 순간 1회)
- **필터**: 하락추세(현재가 < MA)에서는 RSI 과매도여도 **매수하지 않는다**(단일 RSI의 과매도 남발 차단)

**파라미터**

| 키 | 라벨 | 기본값 | 범위 | 의미 |
|----|------|-------|------|------|
| `rsi_period` | RSI 기간 | 14 | 2~999 | 틱봉 `rsi_period`개로 강도(RSI) 계산 |
| `low` | 과매도 | 30 | 1~99 | 상승추세에서 RSI가 이 값을 상향 돌파하면 매수 |
| `high` | 과매수 | 70 | 1~99 | RSI가 이 값을 하향 이탈하면 매도. `0 < low < high < 100` 필수 |
| `ma_period` | 추세 MA | 50 | 2~999 | 틱봉 `ma_period`개 평균선. 현재가가 위면 상승추세로 보고 매수 허용 |
| `bar_ticks` | 틱봉 | 50 | 1~200 | 원시 틱 묶음 크기(클수록 노이즈↓) |

**워밍업**: 50틱봉 × MA50이면 첫 신호까지 약 **2,550틱** 필요. 봇 재시작 시 다시 누적한다.
추세 MA는 느릴수록 필터가 잘 동작한다(너무 짧으면 눌림에 현재가가 MA 아래로 내려가 매수가 막힘).

## 전략 비교

| 구분 | 이동평균 크로스 | RSI + MA 필터 |
|------|----------------|---------------|
| 전략 성격 | 추세 전환 / 추세 추종 | 추세 필터 + 눌림목 반등 |
| 핵심 질문 | "방금 추세가 위로 바뀌었나?" | "상승 추세 중 잠깐 눌렸다가 회복하나?" |
| 매수 신호 | 단기 MA가 장기 MA 상향 돌파 | MA 위에서 RSI가 과매도 회복 |
| 매도 신호 | 단기 MA가 장기 MA 하향 이탈 | RSI 과매수 이탈 또는 MA 이탈 |
| 장점 | 강한 추세 초입을 잡기 좋음 | 추격매수 줄이고 눌림목 진입에 좋음 |
| 단점 | 횡보장에서 가짜 신호 많음 | 강한 급등 초입은 놓칠 수 있음 |
| 잘 맞는 장 | 강한 상승장, 추세장 | 상승장 눌림목, 완만한 우상향 |
| 약한 장 | 횡보장 | 급등장, 강한 하락장 |

> 이 비교표·도움말·파라미터 정의의 **단일 진실 공급원은 `frontend/src/lib/strategy.ts`**이며,
> 대시보드 전략 패널의 도움말 팝업·비교 팝업에 그대로 표시된다. 이 문서와 동기화한다.

## 틱봉 · 확정봉 · 신호 중복 억제

**틱봉(tick bar)**: 원시 틱은 노이즈가 커서, `bar_ticks`개씩 묶은 틱봉의 종가 위에서 지표를
계산한다(`bar_ticks=50` → 50틱봉). `bar_ticks=1`이면 원시 틱과 동일. `to_tick_bars()`가 처리한다.

**확정봉만 평가**: `to_tick_bars`는 최신 틱에 앵커링해 매 틱 재샘플링하므로, 진행 중(미완성)
봉이 매 틱 바뀌어 교차가 경계에서 출렁인다(휘프소 → 신호 폭증). `closed_ticks()`로 길이를
`bar_ticks` 배수로 잘라 **완성봉의 마지막 틱에서만** 샘플링한다 → 휘프소 제거.

**직전 방향 신호 억제(봇 레벨)**: `trading_bot`이 `(종목, 전략)`별 직전 신호 방향을 기억해
**같은 방향 신호를 건너뛴다**. 확정봉 평가와 결합해 **교차 1회당 신호 1건**이 된다.

**미보유 매도 억제**: 봇 매도가능 수량이 0이면 신호 단계에서 주문을 만들지 않는다(상세는
[02-specs 안전 가드](02-specs.md) 참고).

## 지표 — SMA · Rolling SMA · RSI

전략이 쓰는 지표는 모두 **확정 틱봉의 종가열** 위에서 계산한다(`indicators.py`).

### SMA (단순이동평균)

최근 `period`개 종가의 산술평균을 이은 선. 들쭉날쭉한 가격을 부드럽게 펴서 흐름을 보여준다.
구간 길이가 부족한 앞부분(`i < period-1`)은 값이 없다(NaN/None).

### Rolling SMA — 러닝 합계로 구하는 SMA (`rolling_sma`)

매 지점마다 최근 `period`개를 **다시 전부 더하지 않고**, 합계에 새 값을 더하고 창을 벗어난
값을 빼며(**add-new / subtract-old**) **한 번의 순회**로 이동평균을 구한다. 산출값은 일반
SMA와 **완전히 동일**하며(동치성 테스트로 고정), 전체 재합산보다 가볍다.

```python
# rolling_sma(closes, period)  — indicators.py
out = [None] * len(closes)
running = 0.0
for i, value in enumerate(closes):
    running += value                       # 새 값 더하기(add-new)
    if i >= period:
        running -= closes[i - period]      # 창을 벗어난 값 빼기(subtract-old)
    if i >= period - 1:                     # 창이 처음 채워진 지점부터 유효
        out[i] = running / period
```

- 부족 구간(`i < period-1`)은 `None`(SMA의 NaN과 대응).
- **MA 크로스 판정**은 마지막 두 확정봉의 단기·장기 SMA 부호 전환(`short_ma[-2]/[-1]`,
  `long_ma[-2]/[-1]`)만 본다.

> ⚠️ 이름은 "Rolling"이지만 **호출 간 상태를 잇는 증분형이 아니다.** 한 번의 `rolling_sma`
> 호출(=한 틱) **안에서만** 러닝 합계를 쓰고, 다음 틱에는 전체 종가열을 받아 처음부터 다시
> 순회한다. 그래서 함수 1회는 O(m)(m=봉 개수)이지만 **틱당 비용은 O(n)**이다. 호출 간
> 상태를 유지해 틱당 O(1)로 만드는 전환 계획은 [엔진 성능](#지표-계산--엔진-성능) 참고.

### RSI (상대강도지수)

`rsi_period`개 틱봉으로 상승·하락 강도를 0~100으로 나타낸다(`rsi`, pandas 기반). `rsi_ma`
전략에서 과매도(`low`) 상향 회복 = 매수, 과매수(`high`) 하향 이탈 = 매도 판정에 쓴다. 손실이
0이면 RSI=100, 길이 부족 구간은 NaN.

## 지표 계산 · 엔진 성능

### 현재 구조 (채택 — 무상태)

매 틱 `trading_bot.on_tick`이 다음을 수행한다.

```
틱 도착 → 종목 히스토리에 종가 append (최대 _HISTORY_SIZE=6000개 유지)
        → build_strategy()로 전략 객체를 매 틱 새로 생성
        → strategy.evaluate(symbol, 전체_히스토리)   ← 6000개를 통째로 전달
```

`evaluate` 내부에서 `closed_ticks()`+`to_tick_bars()`로 확정 틱봉을 추출하고,
`rolling_sma()`(러닝 합계 add-new/subtract-old 단일 패스)로 이동평균을 구한다.

- `rolling_sma` 1회 호출은 단일 패스라 O(m)(m=봉 개수)으로 효율적.
- 그러나 **매 틱마다 전체 히스토리를 0부터 다시 계산**한다 → 틱당 O(n)(n=히스토리 길이).
  이름은 "Rolling"이지만 호출 간 상태를 잇는 증분형이 아니라 한 호출 안에서만 러닝 합계를 쓴다.

### 왜 현 구조를 유지하는가

현 규모(관심종목 수십 개, 틱 초당 수 회, 히스토리 6000개, 종목당 전략 2~3개)에서 틱당 비용은
**1ms 미만**이라 병목이 아니다. 무상태 구조는 전략 추가·삭제·파라미터 수정 시 상태 수명주기를
관리할 필요가 없어 단순하다(KISS·YAGNI). → **현 구조 유지.**

### 향후 전환 계획 — 틱당 O(1) 상태유지(stateful) Rolling SMA

아래 **트리거 중 하나라도** 발생해 `on_tick`이 실측으로 느려지면 전환한다: 관심종목이 수백 개
규모로 증가 / 틱 빈도 급상승 / `_HISTORY_SIZE`를 수만 단위로 확대.

| 항목 | 현재 | 전환 후 |
|------|------|---------|
| 상태 | 없음(매 틱 전략 재생성) | 봇 레벨에 **종목 × 전략 × 기간(short/long/ma_period)별** SMA 상태 유지 |
| 갱신 | 전체 재순회 O(n) | enqueue + `running += new`, 창 초과 시 dequeue + `running -= old` → O(1) |
| 틱봉 | 매 틱 `to_tick_bars` 재추출 | bar_ticks개가 모이면 한 봉 확정하는 **stateful 누적기** |
| RSI | pandas 전체 재계산 | Wilder 평활의 **증분형**으로 전환 |

전환은 신호 결과를 바꾸지 않는 **순수 성능 최적화**여야 하며(산출값 동치성 테스트로 고정),
봇 재기동 시 상태 워밍업이 필요하다.

## 새 전략 추가 방법

1. `backend/app/strategies/<name>.py`에 `evaluate(symbol, closes) -> Signal | None`를 구현
   (확정봉 평가는 `closed_ticks()` 사용). 파라미터 검증은 `__post_init__`에서.
2. `registry.py`의 `build_strategy`에 식별자→클래스 매핑 추가.
3. `frontend/src/lib/strategy.ts`에 라벨·기본값·파라미터 필드·검증·도움말·비교행 추가(UI 단일 진실).
4. **이 문서(11-strategy.md)에 규칙·파라미터·비교를 추가**하고, 테스트(`tests/test_<name>.py`)를 함께 작성.

> 제거 이력: 단독 `rsi` 전략은 제거되었다(현재 전략 식별자는 `ma_cross` | `rsi_ma` 2종).
