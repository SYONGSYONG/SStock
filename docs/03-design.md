# 03-design.md — UI/UX 설계

## 설계 결정

> 각 결정은 번복하기 어려운 구조적 선택이다. 변경이 필요하면 이 표를 먼저 수정하고 코드에 반영한다.

| # | 선택 | 대안 | 근거 |
|---|------|------|------|
| 1 | **단일 페이지 대시보드 + 탭 영역** | 다중 페이지 라우팅 | 단일 사용자·실시간 모니터링이 목적. 한 화면에서 시세·포지션·로그를 동시에 보는 것이 핵심 |
| 2 | **실시간 데이터는 WebSocket 푸시** | 폴링 | 시세·체결은 지연이 곧 손실. 서버가 대시보드로 WebSocket 푸시, 끊김 시 지수 백오프로 자동 재연결, 관심종목 변경 시 구독 목록 재구성 |
| 3 | **모드 배너 상단 고정** | 설정 안에 숨김 | 모의/실전 혼동이 가장 큰 사고 원인. 실전은 빨강, 모의는 회색으로 항상 노출 |
| 4 | **봇 상태는 명확한 ON/OFF 토글 + 색상** | 텍스트만 | 봇 구동 여부는 즉시 인지돼야 함. ON=초록, OFF=회색, 오류=빨강 |
| 5 | **차트는 lightweight-charts** | 직접 Canvas | 캔들/라인 차트에 최적화, 가볍고 실시간 업데이트 용이 |
| 5a | **차트에 이동평균선(5/20/60/120) + 마우스 시점 tooltip** | 캔들만 표시 | 추세 판단에 이평선이 필수. 마우스 올린 봉의 OHLC·거래량·이평값을 박스로 보여줘 한투 앱 수준의 정보 제공 |
| 6 | **신호와 주문 로그 분리 표시** | 통합 로그 | 신호(전략 판단)와 주문(집행)을 분리해야 디버깅·신뢰 검증 가능 |
| 7 | **라이트 기본 + 다크 토글** | 다크 기본 | 장시간 모니터링 가독성. 선택값 localStorage 저장 |

## 화면 레이아웃 (와이어프레임)

```
┌────────────────────────────────────────────────────────────────┐
│ 모의/실전 토글 …      [봇 시작/정지][시세 시작/정지]  ●모의봇 ●실시간 │ ← 상단 배너(컨트롤은 봇 pill 왼쪽)
├──────────────────────────────────┬─────────────────────────────┤
│ 계좌잔고 (예수금/평가/주문가능…)   │ 오늘 주문한도(횟수/금액/손실) │ ← 2:1
├──────────────────────────────────┴─────────────────────────────┤
│ 보유 포지션 (종목|수량|평단|현재가|손익|손익률)                    │ ← 전체 폭
├─────────────────────────────────────────────────────────────────┤
│ 실시간 시세                                    [🔍 종목검색…]      │ ← 전체 폭
│ 전략이동 | 종목 | 현재가 | 등락률 | 거래량 | 삭제                  │
├──────────────────────────────────┬─────────────────────────────┤
│ 전략: 추가 폼(종목/프리셋/파라미터)  │ 등록된 전략 목록(ON/OFF·추천) │ ← 전략 패널 내부 2열(폼 좌/목록 우)
├──────────────────────────────────┼─────────────────────────────┤
│ 주문내역 (체결/거절)              │ 매매신호 (BUY/SELL·관찰)     │ ← 1:1
├──────────────────────────────────┴─────────────────────────────┤
│ 시스템 로그 (BOT/ORDER/RISK/REGIME/MARKET …)                     │ ← 전체 폭
└─────────────────────────────────────────────────────────────────┘
```

- **상단 배너**: 모의/실전 토글(좌) … **봇/시세 컨트롤(컴팩트)** → **모의봇·실전봇·실시간 pill**(우).
  봇/시세 컨트롤은 모의 봇 pill **왼쪽**에 둔다(`ModeBanner` `controls` 슬롯, `BotControl`/`MarketControl` `compact`).
- **본문(사이드바 없는 세로 흐름)**: 행 단위로 일부는 2열 분할(`.row`).
  1. **계좌잔고 : 오늘 주문한도 = 2:1** (`.row-2-1`)
  2. **보유 포지션** (전체 폭)
  3. **실시간 시세** (전체 폭)
  4. **전략** (전체 폭) — 패널 내부에서 **추가 폼(좌) / 등록된 전략 목록(우)** 2열(`.strategy-body`, 1.4fr·1fr)
  5. **주문내역 : 매매신호 = 1:1** (`.row-1-1`)
  6. **시스템 로그** (전체 폭)
  - 좁은 화면에서는 행·전략 패널 모두 1열로 정렬된다(`.row` ≤1000px, `.strategy-body` ≤1100px).
  - 실시간 시세 표: 헤더 "실시간 시세" + **우측 검색창**, 맨 왼쪽 `전략이동` 버튼, **종목명 클릭 → 차트**, 맨 오른쪽 `삭제`(관심종목 통합).
  - 전략 파라미터 입력은 5열 그리드(`.strategy-params`).
- 포지션 표는 KIS `inquire-balance` 잔고를 원본으로 쓰고, 현재가는 실시간 quote가 있으면 덮어쓴다.

## 색상 기준

### 모드 배너

| 모드 | 색상 | 의미 |
|------|------|------|
| PAPER (모의) | 회색 (`slate`) | 안전, 기본값 |
| LIVE (실전) | 빨강 (`red`) + 굵게 | 실자금 경고 — 항상 강조 |

### 봇 상태

| 상태 | 색상 |
|------|------|
| ON (구동) | 초록 (`green`) |
| OFF (정지) | 회색 (`slate`) |
| ERROR | 빨강 (`red`) |

### 등락/손익

| 의미 | 색상 |
|------|------|
| 상승/이익 | 빨강 (`red`) — 국내 관례 |
| 하락/손실 | 파랑 (`blue`) — 국내 관례 |
| 보합 | 회색 |

> 국내 증시 관례(상승=빨강, 하락=파랑)를 따른다. 색만으로 구분하지 않고 +/- 부호·텍스트 병기(접근성).

## 차트 상호작용 (ChartModal)

캔들차트(일·주·분봉) 위에 **이동평균선**과 **마우스 시점 tooltip**을 얹는다. 한투 앱
차트(`Reference/image/차트화면개선.png`)를 기준 형태로 삼는다.

### 이동평균선

- **5 / 20 / 60 / 120** 4개 라인을 캔들 위에 겹쳐 그린다.
- 계산은 백엔드 `rolling_sma`와 동일한 단순이동평균(SMA). 워밍업 구간(데이터 부족)은
  선을 그리지 않는다(whitespace).
- 색상(범례): 5=주황 `#f59e0b`, 20=파랑 `#2563eb`, 60=초록 `#16a34a`, 120=보라 `#9333ea`.

### 시점 tooltip (crosshair)

`subscribeCrosshairMove`로 마우스가 가리키는 봉을 잡아 HTML 박스로 표시한다(최고/최저
라벨과 같은 오버레이 방식, 가장자리에서 안쪽으로 클램프).

| 행 | 값 | 등락(직전 봉 대비) |
|----|----|------|
| 날짜/시각 | 일·주봉 `YY.MM.DD`, 분봉 `MM.DD HH:mm` | — |
| 시·고·저·종 | 가격(콤마) | 직전 봉 종가 대비 %(부호·색상) |
| 거래량 | 정수(콤마) | 직전 봉 거래량 대비 % |
| 이평 5/20/60/120 | 정수(콤마) | 직전 봉 이평 대비 % |

- **등락 기준은 "직전 봉 대비"**로 통일한다(첫 봉은 비교 대상 없음 → 등락 생략).
- 등락 색상은 국내 관례(상승=빨강, 하락=파랑, 보합=회색)를 따르고 +/- 부호를 병기한다.

## 실시간 갱신 흐름

```
KIS WebSocket → 백엔드 봇 엔진 → (가공/지표 계산) → 대시보드 WebSocket 푸시
  - 체결가/호가: 종목 카드·차트·호가창 갱신
  - 신호 발생: 신호 로그에 즉시 추가 + 해당 종목 강조
  - 주문/체결: 주문 로그 + 포지션 갱신
  - 봇/연결 상태 변화: 상단 배너 갱신
```
