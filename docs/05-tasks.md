# 05-tasks.md — 진행 체크리스트

## 진행 규칙

> 아래 규칙을 위반하면 작업을 중단하고 원인을 해결한 뒤 재진행한다.

1. **순서 엄수** — Phase·단계 번호 순서대로 진행. 건너뜀 금지.
2. **단계별 검증 필수** — 검증 방법을 실제로 실행해 통과를 확인한 후에만 완료(✅) 체크.
3. **테스트 동반** — 각 기능 단계는 테스트와 함께 완료한다(절대규칙 3).
4. **안전 우선** — 실전 주문 경로는 가드 테스트가 통과하기 전까지 활성화하지 않는다.

상태 표기: ⬜ 미착수 / 🟡 진행중 / ✅ 완료

| 4 | KIS 인증 클라이언트 (토큰 발급/캐시/갱신) | mock 테스트 3건 통과 + **실제 모의투자 토큰 발급 확인** | ✅ |
| 5 | 모드 분리 (paper/live 도메인·TR_ID 전환) | 모드별 도메인/TR_ID 단위 테스트 통과 | ✅ |

## Phase 2 — 시세 모니터링

목표: 관심종목 실시간 시세를 대시보드에서 확인.

| # | 단계 | 검증 | 상태 |
|---|------|------|------|
| 1 | 관심종목 CRUD API + DB | CRUD 서비스 테스트 + 라우트(201/409/404) 검증 | ✅ |
| 2 | KIS REST 시세 조회 (현재가) | mock 테스트 2건 + **실서버 현재가 조회 성공** | ✅ |
| 3 | KIS WebSocket 구독 (체결가) + 재연결 | approval_key·체결 파싱·구독 메시지 테스트 통과 (지수 백오프 재연결 구현, 라이브 연결은 장중 통합검증 대기) | ✅ |
| 4 | 대시보드 WebSocket 푸시 | 허브 브로드캐스트/끊긴 클라이언트 제거 테스트 + `/ws/quotes` 등록·해제 검증 | ✅ |
| 5 | 프론트: 관심종목·실시간 시세표 | Vitest 5건 통과 + `vite build` 성공 | ✅ |

> 검증: 백엔드 `pytest` 22건 + 프론트 `vitest` 5건 전부 통과. TestClient로 watchlist CRUD + `/api/quotes/005930` 실서버 200 + `/ws/quotes` 연결 등록(0→1→0) 확인. `vite build` 성공(200KB/63KB gzip).
> React 대시보드: 모드 배너 + 관심종목 CRUD + 실시간 시세표(REST 스냅샷 + WS 병합) + 시세수집 시작/정지. Vite 프록시로 `/api`·`/ws` → 백엔드 :8000.
> 차트(분봉)·호가창은 후속 보강. 봇/시세수집은 안전상 기본 OFF(`/api/market/start`로 수동 시작).

|---|------|------|------|
| 1 | 지표 계산 유틸 (이동평균·RSI, pandas) | 지표 값 단위 테스트 통과 | ✅ |
| 2 | 이동평균 크로스 전략 | 골든/데드 크로스 신호 테스트 통과 | ✅ |
| 3 | RSI 전략 | 과매수/과매도 신호 테스트 통과 | ✅ |
| 4 | 전략 설정 API + 종목별 on/off·한도 | 설정 CRUD 테스트 + 라우트(201/400/404) 검증 | ✅ |
| 5 | 신호 로그 저장 + 대시보드 표시 | 신호 저장/조회 테스트 + 프론트 SignalLog 표시 | ✅ |
