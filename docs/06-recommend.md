# 06-recommend.md — 분야별 추천 종목 (설계 초안)

> 새 페이지 **"분야별 추천 종목"**의 설계 문서. doc-first 원칙에 따라 코드보다 먼저
> 작성한다. Phase 1~5(MVP) 완료 후 추가되는 보강 기능이며, 작업 순서는
> `05-tasks.md`의 **Phase 7**에 기록한다.
>
> **이 문서는 초안이다.** 사용자 검토·확정 후 구현을 시작한다.

---

## 1. 기능 정의 (무엇)

사용자가 **분야(테마)를 선택**하면, 그 분야에 속한 종목들을 **복합 점수로 랭킹**하여
**상위 추천 종목**을 보여주는 페이지.

- 추천은 **정보 제공**이다. 자동 주문과 **분리**한다(전역 안전 원칙). 추천 종목을
  관심종목/전략에 넣는 것은 사용자의 **수동** 행동으로만 일어난다.
- 새 데이터 소스·외부 라이브러리를 추가하지 않는다. **로컬 종목 마스터(.mst) +
  기존 KIS API**만 사용한다(하이브리드).

---

## 2. 분류 체계 (분야 = KRX 테마)

종목 마스터(`종목정보/코스피/kospi_code.mst`, `코스닥/kosdaq_code.mst`)에 이미 들어 있는
**KRX 테마 Y/N 플래그**를 분야로 사용한다(코스피·코스닥 공통 필드).

| 테마 | 마스터 필드 |
|------|-------------|
| 반도체 | `krx_smcn_yn` |
| 바이오 | `krx_bio_yn` |
| 자동차 | `krx_car_yn` |
| 은행 | `krx_bank_yn` |
| 에너지화학 | `krx_enrg_chms_yn` |
| 철강 | `krx_stel_yn` |
| 미디어통신 | `krx_medi_cmnc_yn` |
| 건설 | `krx_cnst_yn` |
| 증권 | `krx_scrt_yn` |
| 선박 | `krx_ship_yn` |
| 보험 | `krx_insu_yn` |
| 운송 | `krx_trnp_yn` |

> 한 종목이 여러 테마에 속할 수 있다(플래그가 독립적). 보통주만 대상으로 하고
> ETF/우선주/관리종목/거래정지 종목은 제외한다(`scrt_grp_cls_code`, `mang_issu_yn`,
> `trht_yn` 필드로 필터).

---

## 3. 추천 점수 (복합 점수)

분야 **내부**에서 각 지표를 0~100으로 정규화(분야 내 순위 백분위)한 뒤 가중합한다.

| 점수 축 | 비중 | 세부 지표 | 데이터 소스 |
|---------|------|-----------|-------------|
| **모멘텀** | 40% | 등락률(20) + 거래량/거래대금 순위(20) | KIS API (`fluctuation` FHPST01700000, `volume_rank` FHPST01710000) |
| **펀더멘털** | 35% | ROE(20) + 영업이익(15) | 로컬 .mst (`roe`, `bsop_prfi`) |
| **수급** | 25% | 외국인(15) + 기관(10) 순매수 | KIS API (`foreign_institution_total` FHPTJ04400000) |

```
종합점수 = 0.40·모멘텀 + 0.35·펀더멘털 + 0.25·수급   (각 축 0~100)
```

- **가중치는 설정값**으로 둔다(코드 수정 없이 조정 — 01-product 페르소나 요구).
- 데이터가 없는 지표는 해당 종목에서 **중립(50)** 처리하고, 추천 카드에 "데이터 없음"
  표시.
- 재무(ROE/영업이익)는 .mst의 **분기 기준일(`base_date`)** 데이터다 → 화면에 기준일 표기.

## 4. 하이브리드 파이프라인 (어떻게)

```
[1] 테마 선택(예: 반도체)
      │
[2] 로컬 .mst에서 krx_smcn_yn='Y' & 보통주 후보 추출   ← API 호출 0
      │
[3] 펀더멘털 점수(ROE·영업이익) 로컬 계산              ← API 호출 0
      │
[4] 펀더멘털 상위 N(기본 30)으로 1차 압축              ← 후보 압축으로 API 절약
      │
[5] 1차 후보에만 KIS 순위/시세·수급 API 조회 → 모멘텀·수급 점수
      │
[6] 종합점수 랭킹 → 상위 K(기본 10) 추천
```

- KIS **rate limit** 대비: 후보 압축(4단계) + 결과 캐시(기본 TTL 5분).
- 장 시간 외/수급 미집계 시: 모멘텀·수급은 중립 처리, **펀더멘털 위주**로 동작(degrade).

## 5. 백엔드 설계

### 파일 (기존 폴더 구조 유지)

| 파일 | 역할 |
|------|------|
| `backend/app/stocks/master.py` (확장) 또는 `stocks/sector.py` (신규) | .mst에서 테마 플래그·재무 파싱 추가 |
| `backend/app/services/recommend_service.py` (신규) | 복합 점수 계산·정규화·랭킹 |
| `backend/app/routers/recommend.py` (신규) | 추천 API 라우터 |
| `backend/app/kis/rankings.py` (신규) | KIS 순위/수급 API 래퍼 |
| `backend/app/schemas/recommend.py` (신규) | 응답 스키마 |

### API (응답 형식 `{"data": ...}` 규약 준수)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/recommend/themes` | 테마 목록 + 각 테마 종목 수 |
| GET | `/api/recommend/{theme}?limit=10` | 해당 테마 추천 종목(종합점수·세부점수·등락률·기준일 포함) |

응답 예시:
```json
{ "data": {
  "theme": "반도체",
  "base_date": "20250331",
  "items": [
    { "symbol": "000660", "name": "SK하이닉스", "score": 87.4,
      "momentum": 92, "fundamental": 80, "supply": 88,
      "change_rate": 3.2, "roe": 18.5 }
  ]
}}
```

## 6. 화면 설계

- **새 페이지** "분야별 추천". 진입 경로는 구현 시 결정:
  - (a) 기존 SPA에 **탭 전환** 추가(의존성 없음, 권장)
  - (b) `react-router` 도입(현재 미설치 — 도입 시 **사용자 승인 필요**, 돌발 의존성 금지)
- 레이아웃: 좌측 **테마 선택**(12개 버튼/칩) → 우측 **추천 카드/표**.
- 카드 구성: 종목명·코드 · 종합점수(큰 숫자) · 세부점수 3축(막대) · 등락률 · ROE.
- 각 추천 종목에 **[관심종목 추가] / [전략 설정으로]** 버튼(수동 연동).
- 03-design.md의 색상 토큰·실시간 흐름 컨벤션을 따른다.

## 7. 안전·제약

| 항목 | 내용 |
|------|------|
| 자동매매 분리 | 추천은 정보 제공. 주문 경로와 무관. 자동 매수 트리거 **금지** |
| 면책 표기 | "투자 판단의 참고용이며 수익을 보장하지 않습니다" 고지 표시 |
| rate limit | 후보 압축 + 캐시로 KIS 호출 최소화 |
| 데이터 신선도 | 재무는 분기 기준일, 수급은 장 종료 후 → 기준 시점 화면 표기 |
| 새 의존성 금지 | 로컬 .mst + 기존 KIS·pandas만 사용. 라우터 도입 등은 사전 승인 |

## 8. 테스트 전략 (테스트 동반 — 절대규칙 3)

- **백엔드**: .mst 테마/재무 파싱, 점수 정규화·가중합, degrade(데이터 없음→중립),
  라우터(mock KIS API)로 `/api/recommend/themes`·`/{theme}` 종단.
- **프론트**: 추천 페이지 렌더, 테마 선택→목록 갱신, 점수 표시, 빈 결과 처리.
- 모든 테스트 설명은 한국어로 작성한다.

## 9. 확정 결정 (구현 기준)

1. 화면 진입: **기존 SPA에 탭 추가** — `react-router` **미도입**(새 의존성 없음).
2. 복합 점수: **모멘텀 40 / 펀더멘털 35 / 수급 25** (설정값, 기본 고정).
3. 1차 압축 **N=30**, 추천 개수 **K=10** (설정값, 기본 고정).
4. 코스피+코스닥 **통합 추천** (시장 구분 없이 한 풀에서 랭킹, 결과에 시장 라벨 표기).
