# 03-design.md — UI/UX 설계 결정

## 설계 결정 8개 표

> 각 결정은 번복하기 어려운 구조적 선택이다. 변경이 필요하면 이 표를 먼저 수정하고 팀 합의 후 코드에 반영한다.

| # | 선택 | 대안 | 근거 | 트레이드오프 |
|---|------|------|------|-------------|
| 1 | **레이아웃: 고정 사이드바 + 메인 컨텐츠** | 상단 탭 내비게이션 / 전체 페이지 이동 | 레포 목록과 현재 위치를 항상 노출해야 관리자가 컨텍스트를 잃지 않음. 14인 팀에서 레포 수가 늘어도 사이드바 스크롤로 수용 가능 | 사이드바가 메인 컨텐츠 가로 폭을 줄임. 칸반 보드처럼 가로 공간이 필요한 화면에서 압박이 생길 수 있어 칸반 진입 시 사이드바 접기 기능 필요 |
| 2 | **칸반 드래그앤드롭: dnd-kit** | react-beautiful-dnd / 드래그 없이 버튼 클릭으로 상태 전환 | react-beautiful-dnd는 유지보수 중단 상태. dnd-kit은 접근성(키보드 조작)·터치 지원이 내장되어 있고 shadcn/ui 생태계와 잘 통합됨 | dnd-kit 학습 곡선 존재. 단, 클릭 방식의 버튼 전환도 병행 제공하여 드래그가 불편한 환경(노트북 트랙패드)을 보완 |
| 3 | **컬러 테마: 라이트/다크 토글 지원, 라이트 기본값** | 다크 모드 기본 / 시스템 설정 자동 따름 | 헤더 우측 토글 버튼으로 즉시 전환 가능. 선택값은 `localStorage`에 저장하여 재접속 시 유지. 사무실 환경·인쇄 공유 시 라이트가 가독성이 높으므로 기본값은 라이트로 설정 | 두 테마 모두 Tailwind `dark:` 클래스로 관리해야 하므로 컴포넌트 작성 시 다크 클래스를 누락하지 않도록 주의 필요. shadcn/ui `ThemeProvider` + `useTheme` 훅으로 구현 |
| 4 | **상태 뱃지: 색상 + 텍스트 병기** | 색상만 / 텍스트만 / 아이콘+텍스트 | 색상만으로는 색각 이상 사용자 구분 불가(접근성). 텍스트만으로는 이슈 목록에서 한눈에 파악이 느림. 색상+텍스트 조합이 가장 빠른 스캔 속도를 제공 | 뱃지 너비가 늘어나 테이블 레이아웃에서 컬럼 폭 조정 필요. 색상 코드는 Tailwind 클래스로 한 곳에 정의하여 일관성 유지 |
| 5 | **이슈 목록: 테이블 뷰 고정** | 카드 뷰 / 테이블·카드 전환 | 14인 팀의 이슈를 빠르게 스캔하려면 행 단위 비교가 용이한 테이블이 적합. 담당자·상태·등록일을 열로 정렬·필터링 가능. 카드 뷰는 정보 밀도가 낮아 이슈 수가 많아질 때 비효율 | 테이블은 모바일에서 가독성이 낮음. MVP는 데스크톱 전용이므로 허용. 추후 모바일 대응 시 카드 뷰 추가 검토 |
| 6 | **칸반 카드 정보 밀도: 요약 (제목 + 담당자 + 레포 태그)** | 최소 (제목만) / 상세 (모든 필드 노출) | 칸반은 진행 상황을 빠르게 파악하는 용도. 제목으로 이슈를 식별하고, 담당자로 책임자를 확인하며, 레포 태그로 출처를 구분하는 세 가지 정보가 최소 필요 단위 | 카드 높이가 일정 수준 고정되어야 보드 전체 높이 예측 가능. 상세 내용은 카드 클릭 시 이슈 상세 페이지에서 확인하도록 유도 |
| 7 | **슈퍼 레포 칸반 통합 뷰: 레포별 색상 테두리 + 레포명 태그** | 레포명 텍스트 태그만 / 레포별 필터 탭으로 분리 | 통합 뷰의 목적은 전체 상황을 한 화면에서 파악하는 것. 필터 탭으로 분리하면 레포를 번갈아 봐야 하므로 목적에 반함. 색상 테두리는 시각적 그룹핑을 즉각 제공하며, 레포명 태그는 색각 이상 사용자를 위한 텍스트 보조 | 레포 수가 많아지면 색상 구분이 어려워짐. N개 레포에 대한 색상 팔레트를 미리 정의하고, 초과 시 패턴(점선·실선 테두리)으로 보완 |
| 8 | **이슈 상세: 별도 페이지 이동 (`/issues/:id`, `/todos/:id`)** | 슬라이드 오버 패널 / 모달 다이얼로그 | 초기에는 슬라이드 오버로 설계했으나, 칸반 등록·담당자 수정·댓글·종결 등 기능이 추가되면서 패널 폭이 부족해짐. 별도 페이지는 충분한 공간과 URL 직접 접근을 제공. 뒤로가기 버튼으로 목록 복귀 | 목록 스크롤 위치를 잃을 수 있음. React Router `key` prop으로 이슈/To Do 간 컴포넌트 재마운트 보장 |
| 9 | **칸반 카드 클릭: 팝업 (읽기 전용) 표시 후 게시판 이동 선택** | 카드 클릭 즉시 게시판 이동 / 인라인 확장 | 칸반에서 드래그 중 실수로 클릭될 경우 페이지 이동을 방지. 팝업에서 핵심 정보를 미리 확인하고 필요시에만 게시판으로 이동 | Radix Dialog 중첩으로 다크 모드 적용 시 `dark` 클래스를 명시적으로 전달해야 함 |
| 10 | **대시보드 레포 클릭: 팝업 (README + 커밋 히스토리)** | 별도 페이지 이동 / 사이드패널 | 대시보드의 목적은 전체 레포 현황 파악. 팝업으로 컨텍스트를 유지하면서 개별 레포 상세를 확인. 커밋 상세도 팝업 내 내비게이션으로 처리 | 팝업 내 다크 모드는 `useTheme`으로 `dark` 클래스를 Dialog.Content에 명시적으로 부여 |

## 색상 코드 기준

> 상태 뱃지(결정 #4)와 레포 태그(결정 #7)에서 공통 사용.

### 칸반 상태 색상

| 상태 | 배경 (Tailwind) | 텍스트 (Tailwind) | 의미 연상 |
|------|----------------|-----------------|----------|
| TO DO | `bg-slate-100` | `text-slate-700` | 중립 — 아직 시작 전 |
| DOING | `bg-blue-100` | `text-blue-700` | 진행 중 — 파란색 활성 느낌 |
| DONE | `bg-green-100` | `text-green-700` | 완료 — 초록색 성공 |
| PENDING | `bg-yellow-100` | `text-yellow-700` | 보류 — 노란색 주의 |

### 레포 구분 색상 (최대 10개 레포 기준)

| 레포 순번 | 테두리 색상 (Tailwind) |
|----------|----------------------|
| 1번 | `border-violet-400` |
| 2번 | `border-pink-400` |
| 3번 | `border-orange-400` |
| 4번 | `border-teal-400` |
| 5번 | `border-cyan-400` |
| 6번 | `border-rose-400` |
| 7번 | `border-lime-400` |
| 8번 | `border-amber-400` |
| 9번 | `border-indigo-400` |
| 10번 | `border-fuchsia-400` |

> 10개 초과 시 테두리를 `border-dashed`로 전환하여 시각적으로 구분.
